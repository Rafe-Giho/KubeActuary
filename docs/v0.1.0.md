# v0.1.0 Goal

v0.1.0 is the first usable alpha release of KubeActuary.

## Release Theme

Local-first evidence-carrying Kubernetes operation capsules.

## Scope

v0.1.0 must support:

- versioned CLI;
- draft local operation capsules;
- inspect capsules;
- attach manual evidence;
- collect `kubectl auth can-i` evidence;
- verify missing or failed evidence;
- open or close an execution gate;
- render a local capsule as a Kubernetes `OperationCapsule` CRD object;
- provide JSON Schema and CRD seed;
- provide examples and tests.

## Non-Scope

v0.1.0 must not include:

- direct execution of proposed cluster writes;
- in-cluster LLM execution;
- controller deployment as a requirement;
- admission webhook;
- external Python runtime dependencies.

## Completion Criteria

v0.1.0 is complete when:

- `python3 -B -m unittest discover -s tests` passes;
- `python3 -B bin/kube-actuary --version` prints `0.1.0`;
- `draft`, `inspect`, `attach-evidence`, `collect auth`, `verify`, `gate`, and
  `render-crd` are usable from the CLI;
- example JSON files parse;
- CRD YAML files parse;
- README documents installation-free local use and the safety boundary.

Current test evidence: [docs/test-results-v0.1.0.md](test-results-v0.1.0.md).
